

## План: AI-чат-бот с консультантом по букетам

Кнопка в правом нижнем углу открывает чат. Пользователь регистрируется (имя, телефон, согласие с политикой), затем общается с ИИ-консультантом, который знает весь каталог. ИИ помогает выбрать букет, считает стоимость, оформляет заказ. При сложностях — создаёт тикет для оператора. Все чаты видны в админке.

---

### Архитектура

**Бэкенд (Lovable Cloud)**
- Включить Lovable Cloud (Supabase) для проекта
- Edge function `chat` — принимает историю сообщений + каталог товаров, отправляет в Lovable AI Gateway с системным промптом флориста-консультанта
- Системный промпт содержит весь каталог товаров (название, цена, описание, наличие) и инструкции: помогать выбирать, считать стоимость, оформлять заказ, при трудностях — передавать оператору

**Хранение чатов (StoreContext)**
- Новый интерфейс `ChatSession` — id, customerName, customerPhone, messages[], hasTicket, createdAt, status
- Все чаты хранятся в StoreContext (как заказы — в памяти)
- Функции: `addChatSession`, `addMessageToChat`, `createTicket`

**Компоненты**

1. **ChatWidget** (`src/components/chat/ChatWidget.tsx`)
   - Плавающая кнопка внизу-справа (иконка бота)
   - По клику — открывается окно чата (slide-up анимация)

2. **ChatRegistration** (`src/components/chat/ChatRegistration.tsx`)
   - Форма: имя, телефон, чекбокс «Принимаю политику конфиденциальности» (ссылка на /privacy)
   - После заполнения — создаётся ChatSession, переход к чату

3. **ChatWindow** (`src/components/chat/ChatWindow.tsx`)
   - Окно сообщений с автоскроллом
   - Поле ввода + кнопка отправки
   - Кнопка «Позвать оператора» — создаёт тикет
   - Защита от дублей: каждое сообщение получает уникальный id, при отправке блокируется кнопка до получения ответа, проверка что сообщение ещё не добавлено

4. **Админка — вкладка «Чаты»** (`src/components/admin/ChatsTab.tsx`)
   - Новая 4-я вкладка в AdminPage
   - Список всех чат-сессий (имя, телефон, дата, есть ли тикет)
   - По клику — просмотр полной переписки
   - Бейдж с кол-вом тикетов

**Edge Function** (`supabase/functions/chat/index.ts`)
- Получает `{ messages, products }` из клиента
- Системный промпт: «Ты — консультант премиального цветочного бренда Магия Роз. Вот наш каталог: [товары]. Помоги клиенту выбрать букет. Считай стоимость. Когда клиент готов — оформи заказ (спроси адрес доставки). Если не можешь помочь — скажи что передашь оператору.»
- Стриминг ответов через SSE

### Защита от дублирования сообщений
- Каждое сообщение имеет уникальный `id` (uuid)
- Кнопка отправки disabled пока ждём ответ ИИ
- Перед добавлением сообщения в массив — проверка что такого id нет
- Ref-флаг `isProcessing` предотвращает повторные отправки
