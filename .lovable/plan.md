## Задача
Добавить в админ-панель функцию автоматической проверки уникальности фото товаров через reverse image search с понятным отчётом по каждому изображению.

## Как будет работать (для пользователя)

В админ-панели появится новая вкладка **«Проверка фото»**:

1. Кнопка **«Проверить все фото товаров»** — запускает массовую проверку
2. Для каждого товара отображается карточка:
   - Превью фото
   - Название товара
   - Статус: 🟢 Уникально / 🟡 Найдены похожие / 🔴 Найдены точные копии
   - Кол-во найденных совпадений в интернете
   - Список ссылок на сайты-источники (с превью и названиями страниц)
   - Кнопка «Открыть в Google Lens» для ручной проверки
3. Можно проверить отдельный товар — кнопка проверки на каждой карточке
4. История проверок сохраняется в БД (дата последней проверки, результат)

## Технический подход

### Backend: Edge Function `check-image-plagiarism`
- Принимает URL картинки (или массив URL)
- Вызывает **SerpAPI Google Lens API** (`https://serpapi.com/search.json?engine=google_lens`)
- Парсит ответ: `visual_matches` (точные/похожие совпадения)
- Возвращает структурированный результат: `{ status, matches: [{title, link, thumbnail, source}], count }`
- Защита: rate-limit (макс 30 запросов/минуту), валидация URL через Zod

### База данных: новая таблица `image_plagiarism_checks`
```
- id (uuid)
- product_id (text, ref к товару)
- image_url (text)
- status (text: 'unique' | 'similar' | 'duplicate')
- match_count (int)
- matches (jsonb) — массив найденных совпадений
- checked_at (timestamp)
```
RLS: только admin-роль может читать/писать (через `has_role`).

### Frontend: новая вкладка `PlagiarismCheckTab.tsx`
- Список всех товаров с фото
- Кнопка массовой проверки (с прогресс-баром: «Проверяется 5/26»)
- Карточки с результатами (как описано выше)
- Загрузка прошлых результатов из БД при открытии

## Проблема: фото в `src/assets/` не имеют публичных URL
Картинки импортированы как модули Vite (`import productX from "@/assets/..."`), и SerpAPI не может их прочитать без публичного URL.

**Решение**: Edge function принимает картинку как **base64**, загружает её во временный Supabase Storage bucket (`plagiarism-temp/`, авто-удаление через 1 час), получает публичный URL и передаёт в SerpAPI. После проверки файл удаляется.

Нужен новый bucket `plagiarism-temp` с публичным доступом на чтение.

## Ограничения
- Бесплатный тариф SerpAPI: 100 проверок/месяц. У вас 26 фото → 1 полный прогон стоит 26 запросов (3 прогона/мес бесплатно).
- Reverse search находит только **то же самое изображение** в сети, а не «похожие концепции букетов». AI-сгенерированные фото обычно дают 0 совпадений — это нормально.
- Проверка одного фото занимает 3-8 секунд (последовательно, чтобы не упереться в rate-limit SerpAPI).
