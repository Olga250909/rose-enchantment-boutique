Добавить в админ-панель отдельный раздел «Заявки» (на оформление праздников) с фильтрами по статусу, отображением деталей и возможностью менять статус.

### Что появится

Новая вкладка **«Заявки»** в админ-панели рядом с «Оформление праздников» (с бейджем количества новых заявок).

В разделе:
- Фильтры по статусу: **Все · Новые · В обработке · Выполнены** (с количеством).
- Список карточек-заявок: услуга, клиент, телефон, дата мероприятия, комментарий, дата создания, текущий статус.
- Селектор статуса в каждой карточке: **Новая → В обработке → Выполнена**.
- Бейдж с количеством новых заявок на самой вкладке (как у «Заказов»/«Чатов»).

Сейчас заявки на оформление падают в общий список «Заказов» — это путает. После изменений они будут идти в свой раздел и не смешиваться с заказами цветов.

### Технические изменения

1. **`src/context/StoreContext.tsx`**
   - Новый тип `DecorRequest`:
     ```
     { id, serviceId, serviceName, servicePrice, customerName,
       customerPhone, eventDate, comment, createdAt,
       status: "new" | "processing" | "completed" }
     ```
   - В контекст: `decorRequests: DecorRequest[]`, `addDecorRequest(...)`, `updateDecorRequestStatus(id, status)`.

2. **`src/components/DecorRequestModal.tsx`**
   - Заменить вызов `addOrder(...)` на `addDecorRequest({ serviceId, serviceName, servicePrice, customerName, customerPhone, eventDate, comment })`.

3. **`src/components/admin/DecorRequestsTab.tsx`** (новый)
   - Фильтры по статусу + список карточек по аналогии с `OrdersTab`.
   - Лейблы: «Новая», «В обработке», «Выполнена».
   - Селектор смены статуса.
   - Пустое состояние «Заявок пока нет».

4. **`src/pages/AdminPage.tsx`**
   - Добавить вкладку `requests` с иконкой `ClipboardList` между «Оформление праздников» и «Доставка».
   - Бейдж с количеством `decorRequests` со статусом `new`.

Существующие «Заказы» и логика чатов не меняются.
