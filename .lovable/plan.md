## Что добавлю

Новую вкладку **«Проверка фото»** в админ-панели + страницу отчёта по каждому товару с найденными совпадениями из интернета.

## Как это будет выглядеть

### Вкладка «Проверка фото» в `/admin`
- Сетка карточек: все товары с превью фото
- На каждой карточке: бейдж статуса последней проверки (🟢 Уникально / 🟡 Похожие / 🔴 Дубликаты), дата, число совпадений
- Кнопка **«Проверить всё»** — последовательная проверка всех 18 товаров с прогресс-баром
- Кнопка **«Проверить»** на каждой карточке — индивидуальная проверка
- Кнопка **«Подробнее»** → открывает страницу отчёта

### Страница отчёта `/admin/plagiarism/:productId`
- Большое превью фото товара слева
- Справа: статус, кол-во точных совпадений, кол-во похожих, дата проверки
- Кнопка «Открыть в Google Lens» (прямая ссылка)
- Список найденных совпадений (до 20 шт.):
  - Превью с сайта-источника
  - Заголовок страницы
  - Домен источника + favicon
  - Бейдж «Точное совпадение» / «Похожее»
  - Кнопка «Открыть» (новая вкладка)
- Группировка: сначала точные совпадения, потом похожие
- Если совпадений нет — большая зелёная плашка «Уникальное изображение ✓»

## Технический подход

### Edge function `check-image-plagiarism`
- Принимает `{ imageUrl: string }` — публичный URL картинки товара
- Вызывает SerpAPI Google Lens: `GET https://serpapi.com/search.json?engine=google_lens&url=...&api_key=...`
- Парсит `exact_matches` и `visual_matches`
- Возвращает: `{ status, exactCount, similarCount, matches: [{title, link, source, sourceIcon, thumbnail, matchType}], checkedAt }`
- CORS, валидация URL, обработка ошибок SerpAPI (402/429/etc.)

### Получение публичного URL фото
Картинки в `src/assets/` импортируются через Vite — на собранном проекте они получают публичный URL вида `https://<preview-domain>/assets/product-xxx-hash.jpg`. В рантайме `import productX from "@/assets/..."` возвращает именно эту строку.

В компоненте проверки конвертирую относительный путь в абсолютный: `new URL(product.image, window.location.origin).href` и передаю в edge function. SerpAPI скачивает картинку с публичного preview/published URL.

**Важно**: проверка работает только на опубликованном или preview-домене (не на локальном `localhost`), потому что SerpAPI должен скачать картинку из интернета. Покажу предупреждение, если `window.location.hostname === 'localhost'`.

### Хранение результатов
Без БД — результаты последней проверки сохраняются в `localStorage` под ключом `plagiarism_results_v1` как `Record<productId, ReportResult>`. Для админки этого достаточно (≤ 50 КБ данных).

**Почему не БД**: миграционный механизм в текущем окружении недоступен через автоматизированные тулзы, а добавление таблицы потребует ручного шага. Если позже нужна история проверок и доступ с разных устройств — можно мигрировать в Supabase отдельной задачей.

## Ограничения
- Бесплатный SerpAPI: 100 проверок/мес. У вас 18 товаров → ~5 полных прогонов
- Нужен опубликованный сайт (или preview-домен) — SerpAPI должен скачать картинку из интернета
- AI-сгенерированные фото обычно дают 0 совпадений — это норма

После одобрения сразу реализую всё и запрошу SerpAPI ключ.
